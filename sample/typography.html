Effective typography communicates our brand, and promotes readability and accessibility.

This system uses scale and color to establish a clear typographic hierarchy. Type must be light and well-spaced to convey transparency and trust in FTB.

Source Sans Pro

Source Sans Pro is our primary typeface for the web. Source Sans Pro is used by CA.gov and is recommended by the U.S. Web Design Standards for its legibility.

Type ramp

The type ramp establishes a crucial design relationship from headings to paragraph text, and ensures a clear and understandable hierarchy between the different levels. This hierarchy builds a structure which enables users to easily scan through content.

  • Source Sans Pro Light is the primary font style.
  • Styles, sizes, and line heights are specified to:
    • Balance content density and reading comfort with usability
    • Enable large type modes for accessiblity
  • For best practices, convert pixels to the relative units, em and rem. They make simple, modular components when used together.

Line length

Comfortable line length allows the user’s eyes to flow easily from the end of one line to the beginning of the next.

For a single column of text, line length must have an average of 90 characters per line, including spaces.

Spacing

Type spacing should be open enough to feel light, but close enough to establish proper readability and appropriate pacing.

Heading followed by text

When a heading is followed by paragraph text, add .25em of space to the established line height.


In this example, an additional 5px (.25em) of space is above the paragraph text.

Text followed by a heading

When paragraph text is followed by a heading, add 1em of space to the established line height.


In this example, an additional 34px (1em) of space is above the Title style.

Heading followed by a subheading

When a primary heading is followed by a secondary heading, add .25em of space to the established line height.


In this example, an additional 8px (.25em) of space is above the Title style.

Paragraph text

When multiple paragraphs are within the same section, add 1em of space to the established line height.


In this example, an additional 20px (1em) of space is between the paragraph text.

Lists

Lists organize written information for users.

  • Apply our custom bullet style to unordered and nested list items.
  • Left-align the first level of list items and only indent the nested items.
  • Add 10px of space above the first level list item.
  • Add 5px of space above the rest of the following list items.

Blockquotes

Blockquotes call out a block of text, as a quotation, from paragraph text.

  • Use sentence case without quotation marks.
  • Add a caption to identify the source of a quotation.
  • For best practices, add a cite atttribute - it specifies the source of a quotation in HTML.

Links

Links lead users to a different page or further information.

  • Source Sans Pro Regular is the font style for text links.
  • Text links are FTB Blue.
  • Visited text links are Grape (purple).
  • Avoid links that open new windows. They cause both accessibility and usability concerns.
    • When they must be used, an accessible tooltip is included to indicate that a new window will open.

Guidelines

Readable text makes good use of alignment, line height, white space, and contrast.

Alignment

Visual elements and columns of type must be left-aligned.

Do. Left-align type to provide the eye a constant starting point for each line. Only indent nested list items.

Don't. Indent the first level of list items.

Line height

Create ample space between lines of type to promote an open feeling and easy read.

Do. Keep Display style short and on a single line in mobile view mode. All other heading styles can wrap to the next line.

Don't. Use Display style if text needs to wrap in mobile view mode.

White space

White space between typographic elements should guide users to focus their attention on the content.

Do. Indicate paragraphs by skipping a line without indentation.

Don't. Fill space with several messages that confuse, distract, or overwhelm users.

Contrast

Create focus points with contrasting type size, color, and style to help users easily scan through content. Ensure colors meet appropriate contrast ratios. See Color.

Do. Use white text on dark backgrounds and Onyx text on light backgrounds.

Don't. Use FTB Blue for text, unless it's for a hyperlink.

 

Do. Use Source Sans Pro Regular in FTB Blue for text links.

Don't.Use all caps, underline, or italic styles for text links.
